## What We Know

- **Firestore stays:** No storage interface for now. The extracted library will be hardcoded to Firestore. Adding a generic storage interface is a future concern.
- **Firestore-specific code moves:** The Firestore scripts, indexes (`firestore.indexes.json`), and related Firebase config that currently live in the Jot repo will move into the new library repo.
- **Separate deploys:**
  - New library repo gets its own dev/prod deploy using its own `.env` files (GCP credentials, project ID, etc.).
  - Jot's deploy is focused purely on Docker image + Cloud Run deployment — no Firebase/Firestore deploy concerns.
- **Primary extraction candidate:** `pkg/memory` — knowledge graph operations, schema, vector search, BFS traversal (added in Plan 1), RRF fusion, deduplication.
- **Secondary candidates (undecided):** `pkg/journal`, `pkg/task`, `pkg/utils` — may or may not move.
