# Inputs
1. ENTRY: The raw text to analyze.
2. ALLOWED PREDICATES: The current canonical predicate ontology.

# Task
Identify clear, factual relationships in the entry and express them as triples.

# Output Format
Return key/value lines only.
status: ok | none
{{- if not .OwnerName}}
owner: <FirstName LastName or empty>
{{- end}}
triples:
Subject | Predicate | Object | SubjectType | ObjectType

# Rules
1. USE NAMES: Use entity names as Subject or Object values.
2. ASSIGN TYPES: For every triple, provide the Type for both Subject and Object (person, place, project, goal, preference, event, asset, tool, object, generic).
3. ONTOLOGY: Prefer predicates from this list: {{.AllowedPredicates}}
4. NEW PREDICATES: If you identify a high-value relationship that clearly does not fit any existing predicate, output it with the prefix NEW: (e.g. NEW:mentors). Only do this for genuinely novel, reusable predicates.
5. LIMIT: Maximum 5 high-confidence triples. If none, set status: none.
6. NO PRONOUNS: Never use first-person pronouns (I, me, my, myself, we, us, our) or second-person pronouns (you, your) as a Subject or Object.
{{- if .OwnerName}}

{{- else}}
7. OWNER DETECTION: If the entry contains a clear self-introduction (e.g. "I'm Alex", "My name is Sarah", "I am Jordan Smith"), output the detected name on the `owner:` line. If no introduction is present, leave `owner:` empty.
